# Frontend Behavior

## Navigation

Every sidebar item routes to a real page. Pages that do not have backend support must show an honest disabled or read-only state instead of fake data.

## Ask CVUM

`/` is the canonical Ask route and opens as an empty chat. `/ask` redirects to `/`. Generation
starts only when the user submits a question or arrives with an explicit `?q=` query parameter.

The Ask page:

- creates or reuses a conversation
- streams SSE events from the backend
- renders the full multi-turn conversation in one continuous thread
- lists real backend conversations in the left chat rail
- opens retrieved source chunks in a right-side drawer
- renders simple markdown formatting
- offers organization spaces: SRE, DevOps, Dev, HR, and Custom
- sends the selected space prompt to the backend as an assistant role instruction
- lets the user choose Knowledge, Web, or Both source mode when those providers are configured
- lets the user choose Fast or Council answer mode
- lets the user configure Council with two response models and one different evaluator model
- posts Helpful and Not Helpful feedback to the backend
- supports copy, Helpful, and Not Helpful actions on persisted answers
- starts a clean conversation on New Chat and can reopen or delete past chats
- waits for attached documents to finish indexing before asking against them
- accepts clipboard images with `Cmd+V`/`Ctrl+V` in the composer and routes them through the same
  visible attachment and OCR indexing flow as the paperclip control
- renders multi-line programs, shell commands, configuration, and queries as fenced code blocks
  with preserved whitespace, a language label, horizontal scrolling, and a copy action
- selects an authorized Project lens and persists the Project on the conversation
- shows an admin-only content-free retrieval trace when tracing is available

Ask no longer uploads or queries a sample runbook automatically.

## Organization Spaces

Ask supports internal organization roles instead of generic consumer categories. Each role changes
the backend role prompt while retrieval and grounding rules stay authoritative:

- `SRE Space`: incidents, reliability, production debugging, logs, metrics, alerts.
- `DevOps Space`: Kubernetes, CI/CD, ArgoCD, Terraform, rollout and rollback work.
- `Dev Space`: code, APIs, databases, implementation plans, debugging.
- `HR Space`: onboarding, policy, access paths, team process.
- `Custom Role`: role prompt generated by the backend LLM from guided questions, then stored in browser local storage for reuse.

## Live Metrics

Dashboard and analytics surfaces read `/api/v1/metrics/overview`. They do not show fabricated numbers or static time series. If historical data is not collected, the UI says so.

## Evals

`/evals` renders the last successful overview and offline benchmark from browser cache immediately.
It then refreshes the live answer observations and slower offline retrieval gate independently in
the background. Refreshing never replaces a valid cached score with a loading skeleton or a false
failure state.

## Knowledge Sources

Confluence, Jira, and Slack cards show live configuration status. Sync buttons call backend sync
endpoints and remain disabled when the connector is not configured. Slack also shows the public
channel allowlist count, connector health, freshness lag, last success, and failures when available.
GitHub repository rows show branch, connector status, last indexed commit/time, and sanitized errors;
the incremental sync control remains disabled until a server-side credential is configured.

Manual local documents are managed through `/documents`.

## Project Lens and knowledge workflows

`/projects` presents a three-step choose/default/verify summary plus source and membership
administration. Project mapping controls relevance; restricted sources still require explicit
grants.

`/incident-copilot` contains three keyboard-navigable tabs:

- Incident Copilot: cited facts, exact-key timeline, status/owner, checks, inference, and gaps.
- Who Knows This?: ranked people with visible evidence signals and scores.
- What Changed?: date-bounded, deduplicated source changes with original links.

Each tab has explicit empty, loading, error, partial, and success behavior. Viewers can use the
workflow and Project selection but cannot access admin mutations.

`/content-health` is the Knowledge Freshness Center. It loads live Project-authorized freshness and
connector data, keeps the last good data visible during a recoverable refresh error, and displays a
bounded top-200 finding list with totals calculated from the full inventory.

## Documents

The Documents page lists documents from all knowledge bases, including Jira, Confluence, and local uploads. It uses backend pagination with large pages, so large connector syncs are browsable instead of capped at the first 100 rows. New manual uploads go to `CVUM Local Uploads` so read-only connector KBs are not polluted.

## Saved Answers

Saved answers are stored in local browser storage. Search, share, and delete actions are functional. This is local user state, not a backend persistence feature.

## Settings

Settings intentionally split two categories:

- Editable local preferences: organization label, locale, theme, and accent color.
- Server-managed runtime settings: auth, SSO, RBAC, connector scopes, RAG weights, provider keys, audit export, rate limits, and cache TTL.

Server-managed rows are read-only, but operational actions with real APIs are wired: test Tavily,
refresh Jira, and refresh Confluence. Runtime cards show the active retrieval weights, source chunk
profiles, comment/attachment indexing state, Council capabilities, auth posture, and eval incidents.

## Access Control

Access Control lists real backend users and role counts. Role changes call the backend admin API. Invite and SSO actions are disabled because those endpoints are not implemented.
