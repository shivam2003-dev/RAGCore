# Kubernetes Deployment Guide

## Building the image
Build the Docker image and push it to our private registry harbor.kimbal.io.
All production images must be signed.

## Creating manifests
Use the service-template repository as a base. Update deployment.yaml,
service.yaml and configmap.yaml with your service values.

## Deploying
Create a pull request in the infrastructure-configs repo. Once approved,
ArgoCD automatically syncs the changes to the production cluster.

## Validation
Check pod status with kubectl get pods -n <namespace> and verify dashboards
in Grafana. For issues ask in #devops-support on Slack.
